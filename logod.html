04 — Guía Completa de Uso

Cómo Usar Cada Elemento del Brand Kit

Instrucciones detalladas paso a paso para implementar correctamente la identidad visual en cualquier contexto.

1

Logo Principal (Horizontal)

El logo horizontal es la versión principal y prioritaria. Úsalo en el 90% de los casos: sitio web, presentaciones, documentos, redes sociales.

✓ Website header ✓ Presentaciones ✓ Propuestas PDF ✓ Banners LinkedIn ✓ Email marketing ✓ Documentos oficiales
  • Fondo recomendado: Siempre sobre #0B0D12 (negro base) o #10141E (dark surface). Nunca sobre azules o púrpuras similares a los del logo.
  • Tamaño mínimo web: 120px de ancho. En móvil puede reducirse a 90px pero no menos.
  • Tamaño mínimo impresión: 4 cm de ancho. Menos de eso, usar el isotipo solo.
  • Zona de seguridad: Dejar espacio vacío equivalente a la altura de la letra "S" en todos sus costados.
  • Escalado: Siempre proporcional. Mantener Shift al escalar en cualquier software.

Errores Más Comunes — Cómo Evitarlos

✗ Lo que NO debes hacer

✗
Usar JPEG/PNG del logo en web
Siempre usar el archivo SVG para web. El PNG es solo para plataformas que no aceptan SVG (email, algunos CMS).
✗
Captura de pantalla del Brand Kit
No recortar el logo de este HTML. Exportar correctamente desde herramienta de diseño (Figma, Illustrator).
✗
Cambiar el tracking del wordmark
El letter-spacing de -2px en el logo es intencional. No alterar nunca.
✗
Favicon sin llaves (solo SC texto)
El favicon SIEMPRE debe incluir las llaves { } como elemento identitario. Ya corregido en v2.1.
✗
Logo sobre fondo de color similar
Nunca el logo azul/magenta sobre fondos azules o púrpuras. Perderá visibilidad y contraste.

✓ Lo que SÍ debes hacer

✓
Exportar SVG desde código fuente
Copiar el SVG del isotipo directamente de este HTML y guardarlo como .svg limpio para uso en Figma/Illustrator.
✓
Fondo oscuro siempre
El sistema dark es la identidad principal. Mantener fondos #0B0D12 o #10141E como base en todos los materiales digitales.
✓
Usar los design tokens en CSS
Copiar los tokens de la sección 03 como variables CSS en tu proyecto para garantizar consistencia automática.
✓
Isotipo con llaves en favicons
Las llaves { } son el elemento más reconocible de la marca. Siempre deben aparecer incluso en 16×16px.
✓
Mantener zona de seguridad
Respetar mínimo 1× la altura del isotipo de espacio libre en todos los costados del logo.

Cómo Extraer los Archivos para Uso Real

1

Extraer el SVG del logo

Abre este archivo HTML en un editor de texto. Busca la etiqueta <svg xmlns="http://www.w3.org/2000/svg" viewBox="0 0 760 230" en la sección HERO. Copia todo el bloque SVG desde esa etiqueta hasta el </svg> correspondiente. Pégalo en un archivo nuevo logo-horizontal.svg.

2

Extraer el favicon corregido (con llaves)

En la sección de Favicon (SEC 1, cuarta card), el primer bloque viewBox="0 0 48 48" es el favicon con llaves + burst. Copiarlo a favicon.svg.

3

Generar PNG desde SVG (para favicons y apps)

Con el archivo SVG listo, usa alguna de estas herramientas gratuitas para generar todos los tamaños PNG necesarios:

# Herramientas online gratuitas: → realfavicongenerator.net — Genera .ico, PNG todos los tamaños, manifest.json → favicon.io — SVG a favicon en 1 click → squoosh.app — Optimizar PNG para web (de Google) → cloudconvert.com — SVG → PNG en cualquier tamaño # Con Inkscape (gratis, desktop): inkscape favicon.svg --export-png=favicon-512.png --export-width=512 inkscape favicon.svg --export-png=favicon-256.png --export-width=256 inkscape favicon.svg --export-png=favicon-128.png --export-width=128 inkscape favicon.svg --export-png=favicon-64.png --export-width=64 inkscape favicon.svg --export-png=favicon-32.png --export-width=32 inkscape favicon.svg --export-png=favicon-16.png --export-width=16
4

Importar en Figma

En Figma: File → Import → selecciona el archivo logo-horizontal.svg. Se importa como frame vectorial editable. Para los colores, crea una librería de estilos con los Design Tokens de la sección 03.

# Colores a agregar como estilos en Figma: SC/Blue #4A8EE8 — CTAs y links SC/Purple #8030C0 — Acentos SC/Magenta #D040E8 — Diferenciador SC/BG/Base #0B0D12 — Fondo principal SC/BG/Surface #10141E — Superficies SC/BG/Card #161B28 — Cards SC/Text/Primary #FFFFFF SC/Text/Secondary #C8D0E0 SC/Text/Muted #7A8AAA
5

Implementar en proyecto web (React / HTML)

/* Opción 1: SVG inline (recomendado — escala perfecto) */ <img src="logo.svg" alt="SingularityCode" width="180" height="auto"> /* Opción 2: Como componente React */ import { ReactComponent as Logo } from './logo.svg'; const Header = () => ( <header> <Logo width={180} aria-label="SingularityCode" /> </header> ); /* Opción 3: Next.js con next/image */ import Image from 'next/image'; <Image src="/logo.svg" width={180} height={54} alt="SingularityCode" />

05 — Guía de Uso

Usos Correctos e Incorrectos

Reglas que protegen la coherencia visual de la marca.

✓ Usos Correctos

✓
Logo sobre fondos dark oficiales
Siempre sobre #0B0D12 o #10141E para máximo contraste e impacto.
✓
Versión mono en impresión de un color
Solo blanco o negro puro. Sin degradados ni colores brand.
✓
Zona de seguridad respetada
Mínimo 1× altura del isotipo de espacio libre en todos los lados.
✓
Favicon con llaves siempre (v2.1)
El isotipo con llaves { } es obligatorio incluso en 16×16px.
✓
SVG para web, PNG para email/doc
SVG escala perfecto en cualquier resolución. PNG para entornos que no aceptan SVG.

✗ Usos Incorrectos

✗
No distorsionar proporciones
Nunca estirar o aplastar. Siempre escalar proporcionalmente.
✗
No recolorear elementos
Solo colores oficiales. No cambiar el azul de llaves ni el magenta del núcleo.
✗
No favicon sin llaves
Las llaves son la firma visual de la marca. Nunca usar solo el burst o solo "SC" como favicon.
✗
No colocar sobre fondos similares
El logo a color nunca sobre fondos azules o púrpuras similares al logo.
✗
No cambiar tipografía del wordmark
Solo Inter 800. No sustituir por otras fuentes ni alterar el tracking.
